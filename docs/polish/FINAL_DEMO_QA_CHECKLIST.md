# Classroom 2.0 — Final Demo QA Checklist

Use this before submitting.

## Build

- [ ] `./gradlew assembleDebug` passes.
- [ ] App launches on emulator/device.
- [ ] No crash on onboarding.
- [ ] No broken icons/resources.
- [ ] No missing permissions.

## One-Device Demo

- [ ] Professor role opens.
- [ ] Student role opens.
- [ ] Role switcher works from both dashboards.
- [ ] Demo mode works without Firebase.
- [ ] No internet required for core demo.

## Attendance

- [ ] Professor starts attendance session.
- [ ] QR displays.
- [ ] Countdown displays.
- [ ] Demo Scan works.
- [ ] Paste payload works.
- [ ] Student receives success confirmation.
- [ ] Points are awarded.
- [ ] Duplicate attendance is blocked.
- [ ] Expired QR is blocked.
- [ ] Professor sees updated attendance count.
- [ ] Attendance history shows record.

## Quiz

- [ ] Professor creates quiz.
- [ ] Empty fields show friendly validation.
- [ ] Student sees active quiz.
- [ ] Student can select answer.
- [ ] Student can submit.
- [ ] Duplicate answer is blocked.
- [ ] Professor sees response count.
- [ ] Professor sees correct percentage.
- [ ] Results bars are readable.
- [ ] Insight dashboard shows recommendation.

## AI Explainer

- [ ] Simple mode works.
- [ ] Example mode works.
- [ ] 3 bullets mode works.
- [ ] Mini quiz mode works.
- [ ] Empty input validation works.
- [ ] Result card looks polished.

## Gamification

- [ ] Points update after attendance.
- [ ] Points update after quiz participation.
- [ ] Correct answer bonus works.
- [ ] Badges appear.
- [ ] Leaderboard is not empty.
- [ ] Current user is visible or highlighted.

## Design

- [ ] Onboarding looks premium.
- [ ] Professor dashboard is screenshot-worthy.
- [ ] Student dashboard is screenshot-worthy.
- [ ] QR screen is clean.
- [ ] Success screen feels rewarding.
- [ ] Quiz results screen is clear.
- [ ] Insight dashboard feels intelligent.
- [ ] AI screen is easy to understand.
- [ ] Leaderboard looks engaging.
- [ ] Dark mode is not broken.
- [ ] No raw error messages.
- [ ] No blank screens.

## README

- [ ] Problem is clear.
- [ ] Solution is clear.
- [ ] Required features listed.
- [ ] Extra features listed.
- [ ] Tech stack listed.
- [ ] Demo mode explained.
- [ ] Demo flow included.
- [ ] Screenshots added or placeholders included.
- [ ] Judging criteria mapping included.

## Final Pitch

- [ ] Demo is under 2 minutes.
- [ ] No step requires typing a long value.
- [ ] No step depends on perfect camera behavior.
- [ ] Presenter knows exact tap path.
- [ ] Closing sentence is memorized.

## Final Submission Standard

The app is ready only when it feels:

- Functional
- Beautiful
- Smooth
- Reliable
- Easy to demo
- Easy to understand
- Better than a normal class project
